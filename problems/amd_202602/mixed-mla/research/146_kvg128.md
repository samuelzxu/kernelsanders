# Attempt 146: kvg=128 for kv>1024

## Hypothesis
Since kvg=64 > kvg=16 for kv>1024 (validated in attempt 145: 88.6µs vs 97.5µs for bs=32),
maybe kvg=128 is even better. Fewer metadata entries = less HBM overhead.

## Changes vs 137
- kv>1024: kvg=128 (was 64)
  - kv=8192/kvg=128: 64 groups per batch item (vs 128 with kvg=64, 2x fewer)
  - Each CTA with nks=32: handles 256 tokens = 2 groups (vs 4 with kvg=64)
- Everything else unchanged (GEMM for small cases, a16w8 for kv<=1024, a8w8 for kv>1024)

## Risk
- 2 groups per CTA might be too few for good kernel efficiency
- The assembly kernel might have internal requirements for >=4 groups per CTA
- If it fails: revert to kvg=64 (approach 137)
