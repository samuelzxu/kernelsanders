# Attempt 160: kvg=32 for kv>1024

## Rationale
kvg=16: too many groups, +9µs (attempt 145)
kvg=64: current best
kvg=128: slightly worse, +1µs (attempt 146)
kvg=32: NEVER TESTED — between 16 and 64

## Analysis
With kvg=32 for kv=8192: 256 groups per batch vs 128 (kvg=64) or 512 (kvg=16)
Each CTA with nks=32: 8 groups per CTA (vs 4 with kvg=64, 16 with kvg=16)
8 groups per CTA might offer better load balancing than 4.
