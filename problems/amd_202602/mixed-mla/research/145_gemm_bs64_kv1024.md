# Attempt 145: Extended GEMM boundary + uniform kvg=16

## Key Changes
1. **GEMM extended to bs<=64/kv<=1024** (was bs<=32/kv<=1024)
   - bs=64/kv=1024: per-XCD KV = 9.4MB fits in L2 (16-32MB)
   - V reads hit L2 on second pass, halving effective HBM reads
   - Avoids assembly overhead (metadata, reduce, dispatch)

2. **Uniform kvg=16** for all assembly paths (was kvg=64 for kv>1024)
   - Reference implementation uses kvg=16 everywhere
   - More fine-grained metadata = better load balancing across CTAs
   - kv=8192/kvg=16: 512 groups per batch (vs 128 with kvg=64)

3. **a8w8 for kv>1024** (CRITICAL: a16w8 is catastrophically slow for long kv)
   - Attempt 143 showed a16w8 regressed bs=256/kv=8192 from 338µs to 627µs

## Expected Impact
- bs=64/kv=1024: 50.5µs → ~30-35µs (GEMM with L2 reuse, big win)
- bs=32/kv=8192: 88.6µs → ~85-90µs (kvg=16 vs 64, uncertain direction)
- bs=256/kv=8192: 338µs → ~330-340µs (kvg=16 effect)
- Geomean: ~77µs → ~72-74µs (driven by bs=64/kv=1024 improvement)
