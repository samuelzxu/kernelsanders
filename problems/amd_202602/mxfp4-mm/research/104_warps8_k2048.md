# #104 num_warps=8 for M=64 K=2048

## Changes

More warps = more in-tile parallelism for memory latency hiding.

## Results (Ranked)
- M=64: 20.0µs (unchanged from #102)
- M=256: 20.1µs (slightly worse, noise)
- Geomean: ~16.0µs (vs #102's ~15.9µs)

## Conclusion
num_warps=8 doesn't help for BSM=32 BSN=64 tiles. The tile is already small
enough that 4 warps can fully overlap memory and compute. More warps add
register pressure without benefit.

Config tuning is exhausted. Revert to #102.
