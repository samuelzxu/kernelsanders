# v102: Inject flydsl stage2 for E=257 bs=512 (FP4 standard path)

## Contrast with v100
v100 tests cktile (BF16 path) for E=257 bs=512.
v102 tests flydsl stage2 injection in the standard FP4 path (no cktile).

## Key Insight from AITER dsv3 CSV
For E=257 bs>=1024, AITER auto-selects `flydsl_moe2_afp4_wfp4_bf16_t64x256x256_reduce`
for stage2. This kernel handles FP4 activations × FP4 weights → BF16 output.

The threshold is at bs=1024, but bs=512 is below the threshold. We can manually
inject this kernel for bs=512 to test if flydsl is actually better.

## Approach
- E=257 bs=512: keep ksplit=0 (standard FP4 quant path)
- Override stage2 to _flydsl_stage2_wrapper with afp4 kernel
- Stage1 uses normal CK FP4 (auto-selected)
- FP4 requant of stage1 output
- flydsl stage2: FP4 × FP4 MFMA → BF16 output

## Why this might help
The AITER-selected CK stage2 kernel for E=257 bs=512 may use suboptimal tiling.
The flydsl kernel is specifically tuned for E=257-like shapes (inter_dim=256).
For stage2 with inter_dim=256 = tile_k, only 1 K iteration is needed (ideal alignment).

## Condition for flydsl injection
- is_flydsl_available() must be True (confirmed in v92)
- inter_dim=256 (E=257 shapes only)
- tokens_per_expert >= 10 (dense enough to benefit)

## Kernel dimensions
- flydsl_moe2_afp4_wfp4_bf16_t64x256x256_reduce
- tile_m=64, tile_n=256, tile_k=256
- For stage2 K=256: exactly 1 K-tile (perfect alignment, maximum efficiency)
- For N=7168: 28 N-tiles
